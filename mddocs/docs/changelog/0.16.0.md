# 0.16.0 (2026-05-12) { #DBR-onetl-changelog-0-16-0 }

## Breaking Changes { #DBR-onetl-changelog-0-16-0-breaking-changes }

### New `.Extra` classes for file connectors { #DBR-onetl-changelog-0-16-0-new-extra-classes-for-file-connectors }

[#484](https://github.com/MTSWebServices/onetl/issues/484)
[#485](https://github.com/MTSWebServices/onetl/issues/485)
[#486](https://github.com/MTSWebServices/onetl/issues/486)
[#487](https://github.com/MTSWebServices/onetl/issues/487)
[#488](https://github.com/MTSWebServices/onetl/issues/488)
[#489](https://github.com/MTSWebServices/onetl/issues/489)

Now all FileConnection classes have `.Extra` classes like this:

- `FTP.Extra(encoding=, use_passive_mode=, ...,)`
- `FTPS.Extra(encoding=, use_passive_mode=, ...,)`
- `HDFS.Extra(timeout=, retry=)`
- `S3.Extra(timeout=, retry=, ssl_verify=, ...)`
- `Samba.Extra(connection_timeout=, ...)`
- `SFTP.Extra(timeout=, host_key_check=, ...,)`
- `WebDAV.Extra(timeout=, retry=, ssl_verify=, ...)`

These new classes allow passing additional params directly to underlying clients:
```python
HDFS(
  ...,
  extra=HDFS.Extra(timeout=..., retry=...),
)
```

Previously some FileConnection classes defined parameters directly in the constructor, e.g.:

- `HDFS(timeout=..., retry=...)`
- `S3(ssl_verify=...)`
- `SFTP(timeout=..., host_key_check=..., compress=...)`
- `WebDAV(ssl_verify=...)`

This is now deprecated.

### Allow passing timeouts to file connectors { #DBR-onetl-changelog-0-16-0-allow-passing-timeouts-to-file-connectors }

[#476](https://github.com/MTSWebServices/onetl/issues/476)

Now developers can pass `timeout` options directly to underlying client. This is a bit different for each connectors - some accept `urllib3.Timeout` object (HDFS, S3, WebDAV), some have explicit attributes for each timeout type (SFTP, Samba).

Thanks to [@NaxNax666](https://github.com/NaxNax666)!

### Allow passing retries for file connectors { #DBR-onetl-changelog-0-16-0-allow-passing-retries-for-file-connectors }

[#476](https://github.com/MTSWebServices/onetl/issues/476)

Now developers can pass `retry` option directly to underlying client of HDFS, S3 and WebDAV connectors, overriding default values.

Thanks to [@NaxNax666](https://github.com/NaxNax666)!

### Change default encoding for FTP & FTPS { #DBR-onetl-changelog-0-16-0-change-default-encoding-for-ftp-ftps }

[#488](https://github.com/MTSWebServices/onetl/issues/488)

Previously these connectors used operation system encoding (e.g. `cp1522` on Windows), now its `utf-8` by default.
